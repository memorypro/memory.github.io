    对上一家公司的一些总结

    发布于 2019-05-27 | 分类于 生活 | 3分钟 | 668字数

    概述

    回家之后换了一个环境，感觉差距有些明显，虽然已经过了三个月了，但是来写这些感觉也不算是很晚，突然有必要对之前的经历做个简单的总结，基本分为以下几块，一次可能写不完，需要分几步来写，第一想说下自己学到了什么，毕竟算上实习也算是一年多了，不能什么都没学到，应该是需要总结以下的，第二部分想简单说一下自己做过的一些项目吧，做个简单的小节，为以后的学习发展什么的提供一些思路，第三部分想说说经历的那些人，那些事，希望不要被前东家的人看到 ::aru:shy::

    学到的技术细节

    这里主要做总结，具体的细节后面会慢慢写的

    1. LoRaWAN
      前东家主要工作就是LoRaWAN终端的代码编写，过程中也了解一些基站，服务器之类的知识，也算是对LoRaWAN的整体架构有一个基本的了解，后面会分几个模块来写这部分内容。
    2. 串口接受的驱动程序
      这些之前自己也思考过，但是始终没有想到一个好的解决方案，有一段时间由于公司结构的调整，有幸和公司嵌入式大牛待着一起，看他们的代码也算是给自己提供了一个良好的思路，这些内容后面会专门写一篇文章来说明这些东西。
    3. 系统监控代码
      自己本来也没有太多的项目经验，只知道做一个产品更多的要注重系统的稳定性，但是后来跟同事学习了一些思路，觉得这个东西比较有用，谁也不能保证自己写出的代码天衣无缝，但是如果你去增加一个监控的任务，去监控系统的功能这个功能比较有效的解决了系统出现死机的状况。
    4. 软件定时器的设计
      这部分内容是在项目过程中自己想出来的，但是并不完善，后面会逐渐的完善这些内容，之前写过一篇文章，但是写的不算完整，并且这部分代码在项目中使用出现了问题，后面应该尽量避免这样的问题。
    5. 还有，得慢慢想

    做过的项目总结

    1. 手持测试节点
    2. 国博的温湿度计
    3. 马来西亚电表
    4. 通用电表模块
    5. 基站通信模块
